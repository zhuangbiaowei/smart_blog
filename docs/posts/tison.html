Apache 基金会治理对话

专访新任 Apache 董事会成员 tison

采访者：庄表伟

引言

在开源世界中，Apache 软件基金会（ASF）作为最具影响力的开源组织之一，其治理模式和运作机制一直备受关注。近期，中国开发者 tison 成功当选 Apache 基金会董事会成员，这是继吴晟、姜宁之后，第三位担任 Apache 董事的中国开发者。本次专访中，tison 详细分享了 Apache 基金会的治理结构、董事会的工作内容以及中国开发者在 Apache 生态中的参与情况。

Apache 基金会董事会是基金会的核心治理机构，负责监督项目运作、审议季度报告，并处理项目中出现的各种问题和争议。

Apache 董事会选举

今年的 Apache 董事会选举竞争激烈，有 20+ 候选人竞争 9 个董事席位。选举过程在每年的成员大会上进行，今年是在 3 月 6 日。

选举流程

  • 提名期

    2 月初开始，持续到会议前一周（2 月 28 日左右）

  • 投票资格

    824 位活跃成员拥有投票权，每人一票

  • 投票期

    持续一周，今年投票率接近 50%（约 400 票）

选举结束后，新一届董事会会在第一次会议上推选主席和副主席。主席主要负责主持各种会议，包括成员大会；副主席则在主席无法履职时接任。

值得注意的是，Apache 董事会会议允许成员作为嘉宾参加，但涉及董事会内部讨论或决议的部分可能会限制在董事会内部进行。

董事会工作内容

Apache 董事会每月第三个周三召开例会，主要工作包括：

审议项目报告

监督 300 多个顶级项目的季度报告，评估项目活跃度和健康状况

处理项目争议

调解项目中出现的冲突，协调不同企业背景的贡献者之间的分歧

项目生命周期管理

决定项目退休、更换项目主席等重大事项，需要董事会决议

跨项目协调

作为中介促进不同项目之间的合作，提供政策澄清和具体支持

董事会会议通常持续 30 分钟到 1 小时 30 分钟不等，取决于议题的复杂程度和争议性。今年的第一次董事会会议吸引了 35 位成员参加，创下历史新高。

基金会治理挑战与转型

作为一个已经发展了25年的基金会，如何进入下一个25年：

低成本运作

每年获得捐赠的规模仅为Linux基金会的百分之一，依赖企业白金赞助（12万美元/年）及基础设施捐赠（如云服务）。

文档危机

历史决策信息分散，老成员隐性知识垄断，威胁治理延续性。tison 计划在任期内系统梳理共识，以书面形式固定并在成员之间、项目当中广泛传播和落实共识。

全球化共识重建

800+成员跨越年龄（20岁至70岁）、文化背景，需重构协作规则以适应多元诉求。

流程革新难题

新生代开发者呼吁改革传统协作方式（如从邮件列表主导转为多种沟通渠道选择），落地时面临现有流程规章的挑战，需要新老成员合作突破。

新兴领域布局

tison透露正推动AI（如机器学习平台）、Rust等前沿技术项目入驻Apache，寻求“标杆项目”突破。

品牌年轻化

尝试自动化工具生成社区周报、AI辅助会议纪要，但实际效果待验证。

中国在 Apache 的参与

虽然 Apache 基金会不强制要求成员提供国籍或公司所属关系，但从可获得的信息来看，中国开发者在 Apache 生态中的参与度正在稳步提升：

董事会代表

从吴晟（2021年）开始，每年董事会中都有一席中国董事，之后是姜宁，现在是 tison

项目贡献

近年来，源自中国的项目数量显著增加

中国行动重点

优化国内项目入孵流程，通过线下见面会、案例手册降低文化摩擦；推动高校开源教育，扩大贡献者基数。

tison 提到，Apache 基金会目前处于活跃期，新项目不断涌入。去年有约 10 个新的孵化项目，今年前三个月已有 5 个新项目加入，显示出健康的发展态势。

结语

通过本次专访，我们深入了解了 Apache 软件基金会的治理结构和运作机制。作为全球最具影响力的开源组织之一，Apache 基金会正在经历新一轮的活跃期，不断有新项目和新成员加入。

中国开发者在 Apache 生态中的参与度也在稳步提升，从董事会到普通成员，中国力量正在发挥越来越重要的作用。随着开源在中国的普及和深入，相信未来会有更多中国开发者和项目加入到 Apache 大家庭中。

开源不分国界，但每个地区的开发者都能为开源社区带来独特的价值和视角。
